# 고객 파이프라인 대시보드 기획

## 1. 문제 정의 (Why)

**현재 상황**: 고객/리드 현황이 Slack 대화, 이메일, 구글시트에 분산되어 있어 CEO가 전체 파이프라인을 한눈에 파악하기 어렵다.

**필요**: 고객사별 현재 단계와 마지막 접촉 일자를 빠르게 조회할 수 있는 단일 뷰.

## 3. 구현 방식

### 아키텍처: 단일 HTML 파일

```
┌─────────────────────────────────────────────┐
│  고객 파이프라인 대시보드                        │
│  ┌──────┐ ┌──────┐ ┌──────┐                 │
│  │ 전체  │ │ 활성  │ │ 주의  │  ← KPI 카드     │
│  │  25  │ │  18  │ │   4  │                 │
│  └──────┘ └──────┘ └──────┘                 │
│                                             │
│  [검색] [상태 필터 ▾] [정렬: 경과일 ▾]         │
│                                             │
│  ┌─────────────────────────────────────────┐│
│  │ 고객사  │ 상태      │ 담당 │ 확인일 │경과│ ││
│  │ CJon   │ 계약서대기 │ 경민 │ 03-17│ 1d│ ││
│  │ LSSC   │ 대규모의뢰 │ 소현 │ 03-17│ 1d│ ││
│  │ CTK    │ 미팅제안중 │ 예란 │ 03-10│ 8d│ ││
│  │ 뉴로퓨전│ 팔로업필요 │ 소현 │ 02-10│35d│ ││
│  └─────────────────────────────────────────┘│
│                                             │
│  [+ 고객 추가] [데이터 내보내기(JSON)]          │
└─────────────────────────────────────────────┘
```

### 기술 스택

- **HTML/CSS/JS** 단일 파일 (서버 불필요)
- **데이터 저장**: HTML 내 JSON 배열로 임베딩
- **수정 방식**: HTML 파일 내 `DATA` 배열을 직접 수정하거나, UI에서 인라인 편집
- **스타일**: 다크 헤더 + 라이트 테이블, 반응형

### 핵심 기능

1. **검색**: 고객사명으로 빠른 검색
2. **필터**: 상태 키워드 필터 (드롭다운 — 고유 상태값 자동 추출)
3. **정렬**: 경과일/고객사명/최종확인일 기준 정렬
4. **색상 코딩**: 경과일 기반 자동 색상 (녹/노/빨)
5. **인라인 편집**: 테이블에서 직접 상태/날짜 수정 → JSON 데이터 자동 업데이트
6. **내보내기**: 현재 데이터를 JSON으로 다운로드 (백업/이관용)

## 6. 성공 기준

- CEO가 브라우저에서 파일 하나로 전체 파이프라인 현황 파악 가능
- 고객사 검색/필터 3초 이내
- 방치 고객 (14일+) 즉시 식별 가능
- 상태 업데이트가 HTML 편집으로 간편하게 가능
